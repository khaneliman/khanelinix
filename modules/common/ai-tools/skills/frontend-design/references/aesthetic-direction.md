# Aesthetic Direction Play

Use before building a new visual surface.

## Decide Direction

Define:

- purpose: what problem UI solves
- audience: who uses it and in what context
- tone: e.g. brutal minimal, industrial utility, editorial, luxury, playful,
  retro-futuristic, art deco, organic, maximalist, or another domain-specific
  direction
- constraint: framework, accessibility, performance, data density
- memorable element: one design idea user will notice

## Visual Rules

- Choose intentional typography; avoid default generic font stacks when starting
  fresh.
- Use cohesive but not one-note color systems. Avoid default purple gradients
  and predictable AI palettes.
- Use CSS variables for repeatable colors, spacing, radius, shadows.
- Match motion to concept; prefer few meaningful transitions over scattered
  animation.
- Create layout with domain-appropriate density. Operational tools should favor
  scanability over decorative hero/card layouts.
- Use texture, imagery, shape, or composition only when it supports product
  context.

## Avoid

- generic centered cards and purple gradients
- stock-like abstract backgrounds when real product/state should be visible
- decorative orbs/blobs
- rounded text pills where icon controls would be clearer
- viewport-scaled font sizes
